## 一、设计背景与原则

在「暴汗艺术家 v3.0」中，我们彻底打通了 **"智能体决策计划"** 与 **"模拟器生理仿真"** 的离散时间闭环反馈。系统不依赖任何写死的假数据，而是通过统一的数据契约（Data Contracts）在每日流转中进行平行宇宙式的演推。

本协议是 **付大佬（数据库 + 仿真器）** 和 **东大佬（LangGraph + Streamlit）** 联调时必须无条件遵守的硬契约。

### 设计原则

| # | 原则 | 反例（禁止） |
|---|---|---|
| 1 | **K & V 字段定死**：字段名、类型、枚举一旦定版不得自由发挥 | LLM 临时编出 `training_type: "cardio"` |
| 2 | **跨系统 ID 必带**：`external_id` + `external_source` 成对出现 | 只存中文动作名，wger 查不到 |
| 3 | **百分比统一**：协议层 0.0-1.0，UI 层 0-100 | 协议层写 `80` |
| 4 | **缺失值统一 `null`** | 空串、`0`、`-1` 混用 |
| 5 | **衍生字段冗余存**：UI 直接读，**不**现场计算 | 前端跑 `actual / target` |
| 6 | **仿真规则优先级显式** | 5 条规则"凭感觉叠加" |
| 7 | **时序数据带时间戳** | 只有数值数组，不知道从哪一秒开始 |

## 二、全局约定

| 项 | 规范 | 反例（禁止） |
|---|---|---|
| **日期格式** | 严格 `YYYY-MM-DD` (ISO 8601) | `2026/6/7`、`06-07-2026` |
| **时间格式** | 严格 `HH:MM` (24h 制) | `23:00 PM`、`晚上 11 点` |
| **布尔 / 枚举** | 统一用字符串枚举，**禁用** int 0/1 | `is_done=1` |
| **百分比** | 协议层统一存 `0.0 ~ 1.0` 浮点；UI 层展示为 `0~100` 整数 | 协议层直接写 `80` |
| **缺失值** | 统一 `null`，**禁用**空字符串、`0`、`-1` | `""`、`0`、`-1` |
| **数值边界** | 超出区间时 `clamp` 到边界（仿真器负责） | 直接溢出 / 抛错 |
| **时区** | 演示用东八区，DB 额外存 `tz_offset` 字段 | 直接用 UTC 戳 |
| **关联键** | 计划 vs 执行统一用 `(user_id, date)` 二元组 | 单独 `date` |
| **外库 ID** | 跨系统调用的字段必须带 `external_id` + `external_source` | 只存中文名 |
| **单位** | 字段名后缀自带单位（`_kg` / `_min` / `_g` / `_pct` / `_bpm`） | 字段名不带单位靠注释 |

### 顶层通用字段（每个协议对象必带）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `user_id` | int | ✅ | 用户主键 |
| `schema_version` | string | ✅ | 当前固定 `"3.1.0"`，用于 LLM 输出 schema 校验 |
| `generated_at` | string | ✅ | `YYYY-MM-DDTHH:MM:SS+08:00`（ISO 8601 + 时区） |
| `source` | enum | ✅ | `"agent"` / `"simulator"` / `"user_input"` 三选一 |

## 六、5 条仿真规则与执行优先级

| 规则 ID | 名称 | 触发条件 | 公式 | 优先级 | 与其他规则的关系 |
|---|---|---|---|---|---|
| **R1** | HRR 恢复乘数 | `training_type = "recovery"` AND `completion_rate ≥ 0.95` | `HRR_rate_multiplier = 1.30` | **P1** | 单独作用 |
| **R2** | 过度训练 HRV 惩罚 | `actual_rpe - target_rpe ≥ 3` | `HRV_{t+1} = HRV_t - 2.5·(RPE_actual - RPE_target) + ε` | **P2** | 与 R1 互斥 |
| **R3** | 降载 + 充足睡眠加分 | `training_type = "recovery"` AND `total_sleep_min > 420` | `Sleep_Score_bonus = +5` | **P3-后** | **在 R5 之后应用** |
| **R4** | 蛋白-深睡增益 | `protein_per_kg ≥ 1.6` | `deep_sleep_{t+1} = deep_sleep_t + 10 + ε` | **P4** | 独立 |
| **R5** | 极度疲劳睡眠崩塌 | `actual_rpe ≥ 8` AND `total_sleep_min < 360` | `Sleep_Score_penalty = -10` | **P3-前** | **在 R3 之前应用** |

> **优先级 P3-前 / P3-后 说明**：R3 和 R5 都影响 `sleep_score`，**必须**先应用 R5 扣分再应用 R3 加分，避免"先加后扣"导致最终 sleep_score 越界。

### 仿真器执行顺序（伪代码）

```python
# Step 1: 取昨日所有态
prev_hrv       = yesterday_execution["cardiovascular"]["hrv_rmssd"]
prev_rhr       = yesterday_execution["cardiovascular"]["resting_hr"]
prev_sleep_scr = yesterday_execution["sleep_execution"]["sleep_score"]
prev_deep      = yesterday_execution["sleep_execution"]["deep_sleep_min"]
prev_weight    = yesterday_execution["body_state"]["weight_kg"]
prev_fat       = yesterday_execution["body_state"]["body_fat_pct"]

# 取今日计划与用户选择
target_rpe   = today_plan["training_plan"]["target_rpe"]
training_t   = today_plan["training_plan"]["training_type"]
planned_prot = today_plan["diet_plan"]["macros"]["protein_g"]
C            = 1.0 if complied else 0.0   # 1=听话, 0=硬练

# 模拟今日实际执行
actual_rpe       = target_rpe if complied else 9
avg_hr           = 110 if complied else 155
peak_hr          = 130 if complied else 175
calories_burned  = 180 if complied else 562
completion_rate  = 1.0 if complied else 0.4
actual_sleep_h   = 7.6 if complied else 5.2
total_sleep_min  = int(actual_sleep_h * 60)

# Step 2: 演推 HRV (R2)
if actual_rpe - target_rpe >= 3:                       # R2 触发
    next_hrv = prev_hrv - 2.5 * (actual_rpe - target_rpe) + ε
else:                                                   # 默认恢复项
    next_hrv = 0.8 * prev_hrv + C * 12 - (1 - C) * 6 + ε
next_hrv = clamp(next_hrv, 25.0, 80.0)

# Step 3: 演推 RHR (常规)
next_rhr = 0.9 * prev_rhr - C * 5 + (1 - C) * 6 + ε
next_rhr = clamp(next_rhr, 55.0, 85.0)

# Step 4: HRR 计算 (R1 乘数)
hrr_multiplier = 1.30 if (training_t == "recovery" and C == 1.0) else 1.0
hr_60s_after   = int(peak_hr - (20 if complied else 12) * hrr_multiplier + ε)
actual_hrr     = peak_hr - hr_60s_after

# Step 5: 演推 Sleep Score (R5 先扣, R3 后加)
R5_triggered = (actual_rpe >= 8 and total_sleep_min < 360)
R3_triggered = (training_t == "recovery" and total_sleep_min > 420)
sleep_penalty = -10 if R5_triggered else 0
sleep_bonus   = +5  if R3_triggered else 0
next_sleep_score = int(
    0.75 * prev_sleep_scr + C * 25 + (1 - C) * 8
    + sleep_penalty + sleep_bonus + ε
)
next_sleep_score = clamp(next_sleep_score, 40, 95)

# Step 6: 演推 Deep Sleep (R4)
actual_protein_g = planned_prot if complied else 64
protein_per_kg   = actual_protein_g / prev_weight
R4_triggered     = (protein_per_kg >= 1.6)
deep_bonus       = 10 if R4_triggered else 0
next_deep_sleep  = int(prev_deep + (15 if complied else -10) + deep_bonus + ε)
next_deep_sleep  = clamp(next_deep_sleep, 40, 120)

# Step 7: 演推 Body Age (用今日 actual_sleep_hours, 不再硬编码)
next_body_age = self.ca + 0.4 * (next_rhr - self.rhr_base) + 1.2 * (7.5 - actual_sleep_h)
next_body_age = round(max(self.ca - 5.0, next_body_age), 1)

# Step 8: 体重与体脂
weight_delta = -0.3 if complied else 0.1
fat_delta    = -0.15 if complied else 0.05
```

| 层 | 命名约定 | 存储位置 | 用途 |
